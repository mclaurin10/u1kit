## F1 — Print Preprocessing dialog compatibility

Detects filament profiles whose `filament_settings_id` lacks an `@Snapmaker U1` suffix — these are likely to trigger Snapmaker Orca's Print Preprocessing rejection. Suggests either rebuilding the profile from a Generic base inside Snapmaker Orca, or using the SD-card workflow (which bypasses Preprocessing). No file-level fix possible.

**Severity:** info.

**Why this matters:** the lineage lives in Orca's profile tree, not in the .3mf. `u1kit` can't rebuild it without modeling Orca's whole profile system. Surfacing this before you send-to-print saves a round trip to a rejection dialog.
