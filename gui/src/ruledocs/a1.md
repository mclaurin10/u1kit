## A1 — Source slicer identification

Detects Bambu Studio, vanilla Orca, Snapmaker Orca, Full Spectrum Orca, or Prusa by inspecting `Metadata/project_settings.config` — printer name, `printer_model`, and the presence of Full Spectrum-specific keys (`mixed_filament_height_*`).

**Severity:** info — no fix; this rule gates downstream rules so they know which slicer produced the file.

**Why this matters:** other rules (B3, D1) behave differently depending on origin. Bambu-origin files need BBL-field stripping; Full Spectrum files need height-bound reconciliation; U1 native files skip most of the conversion work.
